# 今日總覽 UX 優化 — 設計文件

日期：2026-07-22
狀態：已與 User 逐段確認
關係：本文件為主 spec `2026-07-04-active-etf-tracker-design.md` §7① 今日總覽的 UX 增修，不改變資料原則與指標定義。海外代號顯示規則（§4）為全站共用。

## 1. 背景與目標

今日總覽（首頁）上線後累積的 UX 回饋，集中在四個痛點：期間切換看不出是哪天/哪週、切換會跳回頁頂、異動牆一次全列太長、pp 與海外代號顯示不友善。本次針對這些做前端優化，**不動資料層 schema、不新增衍生表、不改指標計算**——全部是呈現層改動。

## 2. 範圍

**納入**（全前端）：
1. 集體動向：期間明確標日期範圍 + 新增「上週/上月」可切換檢視 + 切換不跳頁
2. 異動牆重構：內頁籤（建倉出清/加減碼）+ 市場切換（台股/海外）+ 前 5 筆加「查看更多」+ 說明文字 + N/E/A/T 圖例
3. pp → % 顯示
4. 海外代號去重複顯示（全站共用 helper）

**不納入**：
- 任何 DB schema、pipeline、指標變更
- 海外股中文名（`stock_info` 只有台股，維持 holdings-only）
- 桌面/深色模式以外的新主題

## 3. 集體動向（主 spec §7① 集體動向）

### 3.1 期間按鈕與日期標示

期間選項從 `當日 / 本週 / 本月` 擴充為 **`當日 / 本週 / 上週 / 本月 / 上月`**。**按鈕本身只顯示期間名稱，不顯示日期**；日期資訊由頁面上方單一的「資料更新時間」標示（見 §3.2）承載。

| 期間 | 起訖定義 |
|---|---|
| 當日 | selectedDate 當天（預設為最新交易日，即更新時間那天） |
| 本週 | selectedDate 所在週的**週一 → 週五**（若 selectedDate 落在週間，仍取整週一～五範圍內、且 ≤ selectedDate 的交易日） |
| 上週 | 上一週的週一 → 週五 |
| 本月 | selectedDate 當月 1 日 → selectedDate |
| 上月 | 上月 1 日 → 上月最後一日 |

- 週界定沿用既有 `rangeStartDate` 的「週一為起點」慣例（`(getUTCDay()+6)%7`）；本週/上週以**週一～週五**為範圍（週末非交易日自然無事件，但範圍以工作週界定）
- 上週/上月為新增的「往前一期」查詢：起訖皆早於 selectedDate，資料層 `rangeStartDate` 擴充回傳 `{start, end}`（當日/本週/本月的 end 仍為 selectedDate，上週的 end 為上週五、上月的 end 為上月最後一日）
- 聚合邏輯（`buildCollectiveMovements`、Top 10、排序）不變，只是餵入不同期間的事件集合

### 3.2 資料更新時間標示

頁面上方（首頁 header 區）顯示單一的 **`資料更新至 YYYY-MM-DD`**，值為最新有異動資料的交易日（`availableDates[0]`）。有了這個統一標示，各期間按鈕就不必各自標日期——「當日」即等於更新時間那天，週/月範圍由此推知。若使用者透過日期選擇器切到較早的交易日，此標示仍顯示「資料更新至（最新交易日）」，不隨選擇改變（它代表資料新鮮度，不是目前檢視的日期）。

### 3.3 切換不跳頁

期間切換目前用 `<Link href="/?range=...">` 全頁導航，導致捲動位置重置回頁頂。改為 **`<Link scroll={false}>`**：保留每期間的 server 端資料查詢（App Router 導航），但不重置捲動位置。使用者停在集體動向區塊，內容原地更新。

## 4. 海外代號顯示（全站共用）

### 4.1 問題

海外股票不在 `stock_info`（該表只有台股 twse/tpex/emerging），名稱 fallback 成 stock_id。現有版面 `{stockId} {stockName}` 因此重複顯示：`MRVL US MRVL US`、`00660 KS 00660 KS`。

### 4.2 規則

新增共用 helper `formatStockLabel(stockId, stockName)`：

- 有實際中文名（`stockName` 非空且 `!== stockId`）→ `代號 名稱`，例如 `2330 台積電`
- 無名稱（`stockName` 為空或 `=== stockId`）→ 只顯示代號一次，例如 `MRVL US`、`00660 KS`

放在 `web/lib/format.ts`（既有數字格式工具所在），全站顯示 `代號+名稱` 之處一律改用它：今日總覽異動牆與雷達、ETF 個別頁持股表與時間軸、交集表、個股反查頁。實作時以 grep `{.*stockId.*}.*{.*stockName.*}`（或既有等效寫法）定位所有重複點並替換。

### 4.3 市場判別（供 §5.2 市場切換與去重複共用）

新增共用 helper `stockMarket(stockId): "tw" | "overseas"`：

- 代號結尾為「空格 + 兩個大寫英文字母」（regex `/ [A-Z]{2}$/`，如 ` US`、` JP`、` JT`、` KS`、` KP`、` CH`、` TT`）→ `overseas`
- 其餘（純數字台股代號如 `2330`、`6488`）→ `tw`

註：` TT` 為海外資料源對台股的另一種表示（如 `2330 TT`），歸類為 overseas 是刻意的——它來自全球型 ETF 的海外持股清單，與台股主檔的 `2330` 分屬不同資料脈絡，不合併。

## 5. 異動牆重構（主 spec §7① 異動牆）

以下所有切換（事件類型、市場、查看更多）都是對「選定交易日**已載入到瀏覽器**的那批異動事件」做**客戶端篩選**——按下即改顯示、不重新向伺服器查詢、不跳頁。

> 用詞澄清：這裡的「立即」指前端互動反應快，**不代表資料即時**。持股資料由每日排程（18:30 主場、21:30 補抓）爬各投信當日 PCF，一天一批；PCF 本身一天只揭露一次，盤中不會變動。異動牆呈現的一律是「選定交易日」的當批快照，不是盤中即時買賣。

### 5.1 說明文字與 N/E/A/T 圖例

標題「異動牆」下方加：

- 說明：`列出選定交易日各主動式 ETF 的持股異動事件，可依事件類型與市場切換檢視。`
- 圖例（一行小字）：`新進 NEW＝首次買進｜出清 EXIT＝完全賣出｜加碼 ADD＝增持｜減碼 TRIM＝減持`

### 5.2 兩組切換（button，不換路由）

1. **事件類型內頁籤**：`建倉出清`(NEW+EXIT) / `加減碼`(ADD+TRIM)，預設 `建倉出清`
2. **市場**：`台股` / `海外`，預設 `台股`，用 §4.3 `stockMarket()` 分類

兩組為獨立的 client state，交叉組合（例如「建倉出清 × 海外」）。

### 5.3 前 5 筆 + 查看更多

每個（事件類型 × 市場）組合內：

- 依 `|weightDeltaPct|` 降冪排序（沿用既有 `sortChangeEvents` 的權重次鍵，但主鍵改為僅該組合內權重幅度；建倉出清組內 NEW/EXIT 混排、加減碼組內 ADD/TRIM 混排，皆以權重幅度排序）
- 預設顯示前 5 筆，其餘收合於「查看更多（N）」按鈕，點擊展開全部；再點「收合」還原
- 空組合顯示「此分類當日無異動」

### 5.4 紅漲綠跌

維持既有 `changeTone`/`badgeTone`（NEW/ADD 紅、TRIM/EXIT 綠），不變。

## 6. pp → %（全站數字）

`formatWeightDelta` 從 `+0.0500pp` 改為 **`+0.05%`**（帶正負號、兩位小數，與交集表/輪動的 `+12.33%` 慣例一致）。

- 語意註記：`weight_delta_pct` 本質是權重的「百分點變化」，以 `%` 呈現是刻意的產品決策（使用者偏好），不是相對變化率。此決策與主 spec §6「所有變化以 % 呈現、最多兩位小數」一致
- 邊界：異動事件門檻為權重變化 ≥ 0.05pp，故顯示值最小 `+0.05%`，兩位小數不會塌成 `0.00%`；集體動向的合計權重變化更大，同樣兩位小數
- 套用點：異動牆每筆的權重變化、集體動向的合計權重變化欄

## 8. 測試

- **純函式**（Vitest）：`stockMarket` 各後綴分類、`formatStockLabel` 去重複（有名/無名/名等於代號）、`formatWeightDelta` 的 % 格式與 0.05 邊界、上週/上月起訖計算、異動牆分組排序與前 5 筆切分
- **元件**（Testing Library）：內頁籤切換只改顯示不換路由、市場切換過濾正確、查看更多展開/收合、期間 Link 帶 `scroll={false}`、圖例與說明文字存在、海外列不再重複顯示
- **真資料 smoke**：本機 dev 對正式 Supabase——挑一個有海外持股的交易日，確認 `MRVL US` 不再重複、台股/海外切換筆數合理、pp 全部改成 %
